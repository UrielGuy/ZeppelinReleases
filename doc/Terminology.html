Terminology

This document cover some of the basic concepts in the world of the Zeppeln control software.

Pixel

A pixel is a light source that can be controleld by the software. It can be a single LED on an LED strip, A group of LEDs controlled together (such as a 60 LED but 20 pixel strip), a flood light, a theater light, etc.

Controller

A controller is a piece of hardware that connects to a network (either ethernet or WiFi) on one end, and to some pixels on the other side. At the time of the writing of this document the only well supported contrller is the EasyPixel V2.2 board, made for our software

Scene

A scene is a collection of all of the pixel known to the system, with the knowledge of how they are connected to the setup. It also contains a list of modes and extensions to load and optionally a list of zone divisions.

Mode

A mode is a piece of code that recieves a zone (a set of pixels), looks at their parameters (location in space, how they're connected to the setup, etc.) and outputs their color. A mode may expose its own parameters to control how it makes its decisions.

Zone

A zone is a subset of the pixel in a scene. It could be things like Front, or center, or body. They are defined in the Configuration by a set of rules. Each zone is assigned a mode to control its pixels' colors.

Zone Division

A set of zones. This is what the user selects to say which set of zones is active. Things pairs like Body/Zeppelin or Front/Back

Parameter

A Parameter is a value with a name within the system that can be changed to control the behavior of some element of the system. Parameters can be exposed by modes, extensions, and some are internal to the system. They are controlled and changed by a type of extensions named User Interfaces, such as the OpenGl renderer on screen, the command line user interface, or the web interfce.

Parameters can be of a few types: An integer, a decimal number, a color, a value of an enumeration, a string or a boolean.

Extensions

An extension is a piece of code that extends the capabilities of the software. It could be something to support more functionality. Examples of extensions are the EasyPixel 2.2 driver, the web interface, the power calculations plugin. There are 3 types of extensions (where an extension can satisfy the requirements for more than on type):

  • A user interface - An extensions that gets access to change the values of parameters, hence allowing a user to control the application.
  • A frame consumer - An extenstion that is notified whenever a frame is ready for disaply. An example is the 3D renderer for displaying on the screen or the driver for the EasyPixel controller for sending the frame data over the network.
  • A tick generator - An extensions that notifies the system that it's time to generate another frame. An example is the 3D Renderer which creates a frame whenever the operating system asks for the next frame.

Favorites

A favorite is a set of values in parameters, allowing to save a specific setting in the system and restoring it quickly.
